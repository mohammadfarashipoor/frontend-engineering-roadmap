
# Defining Functions

> How can functions be defined in JavaScript?

---

## 📖 What Does It Mean to Define a Function?

Before a function can execute, it must first be defined.

In JavaScript, there are two primary ways to define a function:

- **Function Declaration**
- **Function Expression**

Although both define reusable behavior, they differ in how JavaScript treats them during execution and where they are typically used.

Understanding these two approaches provides the foundation for many advanced JavaScript concepts, including arrow functions, callbacks, and higher-order functions.

---

## ⚙️ How Does It Work?

A function is first defined.

Once defined, it can be invoked whenever its behavior is needed.

```text
Define Function

↓

Store Function

↓

Invoke Function

↓

Execute Behavior
```

The way a function is defined determines how it behaves before execution and how it can be used throughout the program.

## ⚠️ Common Mistakes

### Assuming Function Declarations and Function Expressions Are the Same

Although both define functions, JavaScript handles them differently during execution.

---

### Choosing the Wrong Definition Style

Use the definition style that best matches the problem instead of treating them as interchangeable syntax.

---

### Focusing Only on Syntax

Understanding when and why each definition style is used is more important than memorizing the syntax.

---

### Writing Anonymous Functions Without a Purpose

Anonymous functions are useful in specific situations, but naming a function often improves readability and debugging.
